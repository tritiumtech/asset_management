# 资产管理系统 - 产品需求文档 (PRD)

**文档版本**: v0.1  
**创建日期**: 2026-04-01  
**文档状态**: 草稿  

---

## 一、背景与目标

### 1.1 现状痛点
- 资产信息分散在多个Excel中，版本混乱
- 领用/归还流程不规范，难以追溯
- 资产状态无法实时掌握
- 盘点耗时耗力，准确率低
- 资产利用率不透明

### 1.2 项目目标
- **短期**: 建立统一的资产台账，实现基础信息化
- **中期**: 领用/归还流程线上化，可追溯
- **长期**: 资产利用率分析，支撑采购决策

## 二、需求范围

### 2.1 资产类型

| 类别 | 示例 | 管理重点 |
|------|------|----------|
| IT设备 | 笔记本、显示器、服务器 | 序列号、保修期、配置 |
| 网络设备 | 交换机、路由器、AP | 位置、IP、维保 |
| 办公设备 | 打印机、投影仪 | 位置、耗材、维修记录 |
| 软件资产 | License、订阅服务 | 数量、有效期、分配 |
| 配件耗材 | 键盘、鼠标、线缆 | 库存、领用记录 |

### 2.2 功能需求

#### 2.2.1 资产台账管理

**基础信息字段**:
- 资产编号（唯一标识）
- 资产名称
- 资产类别
- 品牌/型号
- 序列号/SN码
- 购置日期
- 购置金额
- 保修期
- 供应商
- 存放位置
- 使用状态（库存/在用/维修/报废）
- 当前使用人
- 所属部门
- 备注

**扩展信息**:
- 配置参数（JSON格式，灵活扩展）
- 附件（发票、保修卡照片）
- 关联资产（如笔记本+显示器套装）

#### 2.2.2 资产流转流程

**领用流程**:
```
申请人提交 → 部门负责人审批 → 资产管理员确认 → 领用完成
```

**归还流程**:
```
使用人申请归还 → 资产管理员验收 → 检查资产状态 → 归还入库
```

**调拨流程**:
```
调出方申请 → 双方负责人审批 → 资产管理员确认 → 调拨完成
```

**报废流程**:
```
使用人/管理员申请 → 部门负责人审批 → 财务确认 → 报废处理
```

#### 2.2.3 查询与统计

**查询能力**:
- 按资产类别筛选
- 按使用状态筛选
- 按使用人/部门筛选
- 按位置筛选
- 关键字搜索

**统计报表**:
- 资产总览（数量、金额）
- 类别分布
- 部门分布
- 状态分布
- 即将过保资产预警
- 资产利用率分析

## 三、技术方案

### 3.1 方案选型：飞书生态原生

| 组件 | 选型 | 说明 |
|------|------|------|
| 数据存储 | 飞书多维表格 | 灵活扩展，权限可控 |
| 审批流程 | 飞书审批 | 内置流程引擎，消息通知 |

### 3.3 飞书审批流程设计

**审批模板：资产领用申请**
- 申请人（自动）
- 申请日期（自动）
- 资产名称（下拉选择，关联资产台账-库存状态）
- 资产编号（自动带出）
- 领用事由
- 部门负责人审批
- 资产管理员确认
